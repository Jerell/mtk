1  ModelingToolkit.jl

Modified

April 30, 2024

ModelingToolkit.jl is a modeling framework for high-performance symbolic-numeric computation in scientific computing and scientific machine learning. It allows for users to give a high-level description of a model for symbolic preprocessing to analyze and enhance the model. ModelingToolkit can automatically generate fast functions for model components like Jacobians and Hessians, along with automatically sparsifying and parallelizing the computations. Automatic transformations, such as index reduction, can be applied to the model to make it easier for numerical solvers to handle.

1.1 Syntax

Components are defined as sets of parameters, variables, subcomponents/systems, and equations. These are passed into functions like NonlinearSystem and ODESystem that construct the system of equations.

1.1.1 Systems and solving

A system is used to create a problem which is then solved.

@named nlsys = NonlinearSystem(eqs, vars, pars)
sys = structural_simplify(nlsys)
prob = NonlinearProblem(sys, [], []) # [initial conditions], [parameters]
@named odesys = ODESystem(eqs, t, vars, pars)
sys = structural_simplify(odesys)
prob = ODEProblem(sys, [], (0.0, 0.0001), [])
sol = solve(prob)

These systems can also be defined using macros.

pars = @parameters F=100 d=1 k=1000
vars = @variables x(t)=0.0 ẋ(t)=F/d ẍ(t)=0.0
eqs = [
1    D(x) ~ ẋ
    D(ẋ) ~ ẍ
    d*ẋ + k*x^1.5 ~ F
]
@mtkbuild odesys = ODESystem(eqs, t, vars, pars)
1
~ is used in place of \(=\) because = is already used for variable assignment in code, but here we want to define a symbolic equation

1.1.2 Components

Components can be written using an explicit function syntax, where the function returns the system, or using macros.

@connector function FlowPort(; name)
    sts = @variables begin
        p(t) = 1.013e5
        (qm(t)=1, [connect = Flow])
    end
    ODESystem(Equation[], t, sts, []; name=name)
end
@connector Port begin
    p(t)
    ṁ(t) = 0, [connect = Flow]
end

1@mtkmodel Orifice begin
    @parameters begin
        C₀=2.7
        A₀=0.00094
        ρ₀=1000
        p′=0
    end
    @variables begin
        ṁ(t)=0
        p₁(t)=p′
        p₂(t)=p′
    end
    @components begin
        port₁ = Port()
        port₂ = Port()
    end
    @equations begin
        ṁ ~ +port₁.ṁ
        ṁ ~ -port₂.ṁ
        p₁ ~ port₁.p
        p₂ ~ port₂.p

        p₁ - p₂ ~ (1/2) * ρ₀ * u^2 * C₀
    end
end
1
the @mtkmodel macro is equivalent to the function that returns an ODESystem in the prior code block

1.2 Variables

After constructing a system, ModelingToolkit identifies unknown and observed variables and equations.

Observed equations look something like this

observed(odesys_x)
15-element Vector{Symbolics.Equation}:
 xˍt(t) ~ 6.283185307179586amp*f*cos(6.283185307179586f*t)
 xˍtt(t) ~ -39.47841760435743amp*(f^2)*sin(6.283185307179586f*t)
 xˍttt(t) ~ -248.05021344239853amp*(f^3)*cos(6.283185307179586f*t)
 x(t) ~ x₀ + amp*sin(6.283185307179586f*t)
 ẋ(t) ~ xˍt(t)
 ẋˍt(t) ~ xˍtt(t)
 ẋˍtt(t) ~ xˍttt(t)
 ẍ(t) ~ ẋˍt(t)
 ẍˍt(t) ~ ẋˍtt(t)
 p(t) ~ (-M*g - M*ẍ(t)) / (-A)
 pˍt(t) ~ (M*ẍˍt(t)) / A
 r(t) ~ r₀*(1 + p(t) / β)
 rˍt(t) ~ (r₀*pˍt(t)) / β
 ṙ(t) ~ rˍt(t)
 ṁ(t) ~ A*r(t)*ẋ(t) + A*x(t)*ṙ(t)
Note

Some of the observables have a ˍt appended to the name. These are called dummy derivatives, which are a consequence of the algorithm to reduce the system DAE index.

2 Bifurcation diagrams

Bifurcation diagrams describe how, for a dynamic system, the quantity and quality of its steady states changes with a parameter’s value. These can be computed through the BifurcationKit.jl package. ModelingToolkit provides a simple interface for creating BifurcationKit compatible BifurcationProblems from NonlinearSystems and ODESystems.
